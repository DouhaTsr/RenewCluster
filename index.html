À propos du Projet

Le projet consiste à mettre en place un cluster utilisant des machines équipées des systèmes d'exploitation Windows et Linux, intégrant les technologies clés telles que Hadoop, Spark, YARN et HDFS. L'objectif est de créer une infrastructure distribuée capable de gérer de grandes quantités de données de manière efficace et performante.

Technologies Utilisées

Hadoop

Hadoop est un framework open-source qui permet de stocker et de traiter de grandes quantités de données sur des clusters de serveurs. Composants principaux : Hadoop Distributed File System (HDFS) : Un système de fichiers distribué conçu pour stocker de grandes quantités de données de manière fiable sur des machines à faible coût. MapReduce : Un modèle de programmation et un système de traitement par lots pour le traitement parallèle de données volumineuses.

Yarn

YARN est le gestionnaire de ressources de Hadoop. Il permet de gérer et de planifier les ressources informatiques (comme la mémoire et la puissance de calcul) sur un cluster Hadoop. Il divise les fonctionnalités de gestion des ressources et de planification de tâches qui étaient initialement intégrées à MapReduce dans les versions précédentes de Hadoop. Il est utilisé pour déployer divers frameworks de traitement de données, tels que MapReduce, Spark et d'autres applications compatibles avec Hadoop, en optimisant l'utilisation des ressources du cluster.

Spark

Spark est un framework de traitement de données en mémoire rapide et évolutif, également open-source. Il Permet de stocker temporairement les données en mémoire pour un accès plus rapide, contrairement à MapReduce qui écrit et lit des données sur disque. Il est particulièrement adapté aux applications nécessitant un traitement rapide des données et des itérations fréquentes sur les ensembles de données, offrant une alternative plus rapide et plus flexible à MapReduce.
